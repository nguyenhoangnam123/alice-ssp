# SSP — Self-Service Portal

Closes the gap between vibe-coding product engineers and platform engineers. A product
engineer describes a service in a paragraph; Bedrock Claude Opus 4.6 validates, rejects
or generates Dockerfile / CI workflow / Helm values / ArgoCD Application; a real PR
opens against this repo for a human to merge; ArgoCD reconciles into EKS.

## Layout

```
alice/
  llm-product-poc/                 # Portal: Next.js 15 + Drizzle + Postgres
  fleet-managers/
    terraform/foundation/00..70/   # DevOps-owned: bootstrap → portal-app
    terraform/modules/             # Per-tenant: namespace, IAM
    helm/app/                      # Generic Helm chart for any service
    argocd/                        # App-of-Apps root + ApplicationSets
    platform-apps/ssp-portal/      # Helm values for the portal itself
    tenants/<domain>/apps/<svc>/   # Per-service values + Dockerfile + CI + ArgoCD app
                                   # (generated by the AI, reviewed by humans)
  docs/                            # See "Read first" above
  .github/workflows/               # CI + ECR build via GitHub OIDC
  .kubeconfig                      # Isolated cluster config (gitignored)
```

## Status

All foundation layers live in this account (`195748744911`, eu-west-1). The portal
serves real traffic on real DNS with a real TLS cert behind a WAFv2 ACL. Bedrock Opus
4.6 is wired through IRSA to the portal pod. Every approved ChangeRequest opens a real
GitHub PR against this repo. See [docs/e2e-evidence.md](./docs/e2e-evidence.md) for the
receipts.

## Quick start (operator)

```bash
# Cluster access (isolated kubeconfig — won't clobber your other clusters)
export KUBECONFIG=$PWD/.kubeconfig
kubectl get nodes

# Tail the AI agent live
kubectl -n ssp-portal logs -f deploy/ssp-portal-app

# Submit a CR as the seeded admin
USER_ID=a229f8aaaa694e1b865e76f820
curl -X POST https://portal.ssp.mightybee.dev/api/services \
  -H 'Content-Type: application/json' -H "Cookie: ssp_session=$USER_ID" \
  -d '{"tenant_id":"84b02a25117e4b7fbaa1983b64","name":"demo","subdomain":"demo",
       "vpn_internal":false,"git_repo":"https://github.com/ORG/sample",
       "description":"A small public web service for the growth team."}'
```
